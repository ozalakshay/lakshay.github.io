GitHub

Project Overview

This project involved deploying a simple quote generator web application using Node.js, HTML, CSS, and JavaScript on a Google Cloud Compute Engine virtual machine. It demonstrated full-stack deployment and infrastructure setup using both manual CLI and Infrastructure as Code (IaC) principles with Terraform.

Key Features

  • Created and configured a VM instance using gcloud CLI and automated the same process using Terraform
  • Installed Node.js, cloned the application repository via SSH, and launched the Express.js app on port 3000
  • Defined and applied GCP firewall rules to allow HTTP traffic to the VM instance
  • Managed background service execution and validated application availability through the VM's public IP
  • Used terraform apply to provision infrastructure and terraform destroy for clean-up

Technologies Used

Cloud Platform: Google Cloud Platform (Compute Engine)
Infrastructure as Code: Terraform
Backend Stack: Node.js, Express.js
Frontend: HTML, CSS, JavaScript
Tools: gcloud CLI, SSH, Linux (Debian), Git
Networking: HTTP firewall rules, public IP routing

Challenges & Solutions

One of the key challenges was automating infrastructure setup to be reusable and configurable. This was addressed by using Terraform to define infrastructure as code, ensuring consistency and ease of deployment. Another challenge was securing and exposing the app publicly, which required correctly configuring firewall rules to allow inbound traffic on the required port.

  • Configured GCP's Ingress to expose the application to the public at http://go-web.local

Technologies Used

Language: Golang
CI/CD: GitHub Actions, ArgoCD
Containerization: Docker (Dockerfile)
Orchestration: Kubernetes (GKE)
GitOps: ArgoCD
Infrastructure: GCP (GKE, Ingress, Cloud Shell)
Dev Tools: Git, Docker Hub, YAML, K8s manifests

Hands-On Experience

  • Wrote and optimized a production-ready Dockerfile for the Go app
  • Created and applied Kubernetes Deployment, Service, and Ingress YAMLs
  • Used kubectl to manage cluster resources and troubleshoot deployment issues
  • Set up GitHub Actions workflow with custom YAML for CI pipeline
  • Deployed and monitored the app using the ArgoCD web UI

Project Overview

Deployed the full-featured Online Boutique microservices application provided by Google Cloud to understand how modern, large-scale distributed systems are structured and managed in production environments. This demo runs on Google Kubernetes Engine (GKE) and showcases gRPC-based service communication, horizontal scaling, and cloud-native observability.

Hands-On Experience

  • Created a GKE Autopilot cluster and applied Kubernetes manifests to deploy 11 interconnected microservices
  • Explored how each microservice communicates over gRPC and discovered how service orchestration is handled
  • Examined real-time metrics and logs through kubectl logs and GCP's Cloud Operations tools
  • Analyzed service dependencies, crash recovery, and how Redis, gRPC, and JSON-based product catalogs are used in production
  • Used kubectl and gcloud CLI for deployment, monitoring, and cluster cleanup

Architecture Highlights

The app includes services written in Go, Python, Node.js, Java, and C#. They simulate real-world features like checkout, product recommendations, email notifications, currency conversion, and ads. All services communicate using Protocol Buffers and gRPC, with Redis used as an in-memory store.

  • Frontend (Go) serves the web UI
  • Cart, checkout, shipping, and payment services mimic full e-commerce functionality
  • Ad and recommendation engines enhance personalization
  • Email service and load generator simulate post-purchase flow and traffic

Cloud Tools & Technologies Used

Kubernetes (GKE), gRPC, Docker, Redis, Cloud Monitoring, Cloud Logging, Git, Protocol Buffers, GCP Ingress, Spanner (optional variant)

Outcome

This project helped me understand how microservices are deployed and managed on cloud-native infrastructure, how services interact in real time, and how to investigate issues via centralized logging and monitoring. It gave me hands-on insight into infrastructure that closely resembles enterprise production systems.
